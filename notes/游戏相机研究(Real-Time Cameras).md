## 0. 为什么相机难
玩家**透过相机感知整个世界**——相机一抖、一晕、一丢目标，体验立刻崩。3C 里相机是把"角色能力 + 控制输入 + 场景"翻译成"一块好看好懂的画面"的中间层，约束多、主观性强，所以有一整本书。

## 1. 通用心智模型：Desired → Smoothed
几乎所有游戏相机都是两步：
1. **算"期望状态"**：期望位置（Desired Position）+ 期望朝向（Desired Orientation / look-at）。
2. **平滑地逼近它**：不要瞬间跳到期望值，而是用**阻尼 / 弹簧 / 滞后(lag)** 平滑过去。

"算期望"决定相机想去哪；"平滑"决定它怎么去、是否跟手、晕不晕。**大多数手感问题出在第 2 步的参数，或第 1 步选错了参考系（贴车头 vs 贴速度方向）。**

## 2. 第三人称 / 追尾相机解剖
- **Look-at position**：相机看向的点（常不等于角色原点，可加偏移/抬高）。
- **Desired position**：通常在角色**后上方**，沿"前进方向"偏移（boom/臂）。
- **Boom / SpringArm**：从挂点伸一根臂到相机；可带碰撞回缩。
- 关键张力：**贴得紧**（朝向锁定角色）→ 跟手但角色一转整个屏幕跟着甩、晕；**带滞后**→ 自然但角色会在屏幕里漂、甚至跑出画面。**好相机在两者间找临界点。**

## 3. 重定向（Reorientation）——朝向怎么平滑转（书 Ch7）
书里把"相机怎么转向期望朝向"列了几档，从差到好：
1. **Constant angular velocity**（恒定角速度）：实现简单，但起止都**突兀**。
2. **Acceleration / deceleration**：有平滑起止，但减速不够会**过冲**。
3. **Angular velocity damping**（角速度阻尼）：比例阻尼，平滑起止，**推荐基础**。
4. **Reorientation lag**（重定向滞后）：故意给朝向一点延迟 → 更"自然"，还能让玩家**看到前方**；但"朝向滞后 = 角色在屏幕里偏移"，要和"保持角色在画面内"平衡。**完全无滞后**的后果：相机死盯角色、看不到前方，且**任何急转或被撞都让整个屏幕猛甩**。
5. **Springs / PID**：用弹簧或反馈控制器把"当前角度→期望角度"的差平滑收敛。**必须临界阻尼(critically damped)**，否则会绕着目标振荡。调到临界很费时间但非常值。

> ⭐ **甩尾感 vs 车头转向感**：本质就是这里的滞后量。滞后大/松 → 相机跟不上车头 → 看到车身在画面里斜转 = **甩尾感**；临界阻尼且跟得紧 → **车头始终朝前、世界绕着转** = 车头转向感。

## 4. 运动方向 vs 物体朝向（赛车相机的关键，书 Ch5）
书里"Racing games → Ground vehicles"专节原文要点：
- 载具相机喜欢待在**前进方向的后上方**；位置和朝向**都用 lag**，让车的朝向能相对相机变化（强调立体感）。
- **位置滞后 ⇒ 车在屏幕上会移动 ⇒ 必须保证车始终至少部分在画面内。**
- ⭐ **当朝向剧烈变化时（碰撞、急转、翻滚），用车的"运动方向(速度)"而不是"车头朝向"来决定相机位置，并在失控时加额外阻尼。**

这条是治我们两个病的钥匙：
- **甩尾**：别死贴车的 transform yaw，而是朝**速度/前进方向**做带阻尼的重定向。
- **倒车出画**：见 §6。

## 5. 预测式相机（书 Ch2 / Ch7）
- **Predictive vs reactive**：预测式相机能提前知道目标要去哪，导航更从容。
- **Target position prediction**：不必精确预测，**用"角色运动意图"**即可——常见做法：look-at 点沿运动方向**提前一段固定距离+抬高**，让玩家看到前方。代价：角色急转时 look-at 猛动 → **要对它加阻尼**。

## 6. 弹簧与"目标朝相机移动"（倒车问题，书 Ch9）
- **过冲**常发生在**比例控制 + 目标朝相机移动 / 突然停下**（缩短相机-目标距离快过相机能减速）→ 相机绕目标振荡。
- 解法：**弹簧 + 临界阻尼**（无过冲地收敛）。弹簧**别太硬**，否则相机像根杆子焊在车上（"rod-like"），失去"有机"感。
- **Movement toward the camera**（角色倒着把相机怼进墙/拉近）：可让相机沿碰撞面滑动（保持碰撞体简单），或用"运动方向 × 表面法线"叉积判断该往哪侧让。

> ⭐ **倒车出画**就是"目标朝相机移动"：车后退 = 朝身后的相机怼过去。处方：① 临界阻尼的位置弹簧防过冲；② 倒车时用**速度方向**重新定位相机（车往哪开相机就去哪后面），或倒车时**把臂拉长/抬高**保证车在画面里。

## 7. 相机碰撞 / 遮挡（书 Ch8、Ch9）
- 相机本质是个会**射线/体积探测**的 AI 物体：被墙挡住目标(遮挡)要回缩/穿避；钻进狭缝要处理。
- 设计上：碰撞体**尽量简单、倒角**，给相机留足空间，单独的相机碰撞 mesh。
- 解析顺序（Collision resolution）：不动 / 部分移动 / 改走别处 / 跳切 / 必要时禁用碰撞。

## 8. 数学工具箱（书 Ch10）
- **Spring（弹簧-阻尼）**：`F = -k·x - c·v`，临界阻尼 `c = 2√(k·m)`，无过冲最快收敛。相机位置/角度平滑都可用。
- 朝向插值：角度插值 vs 目标位置插值；四元数 slerp；注意**共线/相反朝向**的退化、gimbal lock、roll 累积。
- 数字滤波（低通）平滑噪声；样条(Catmull-Rom 等)做轨道相机。

## 9. 我们的实践（Sunday Drive）

### 现状
- 玩家车 = `ASimpleCarPawn`，相机用 UE 的 `USpringArmComponent`(后上方，臂长 500~700) + `UCameraComponent`，开了位置 lag 与旋转 lag、继承 Yaw。
- Rat 车型转向猛(轴距短)，把相机问题放大。

### 用书诊断出的两个病
1. **转向"甩尾感"**(§3/§4)：`CameraRotationLagSpeed` 调小(松) → 旋转滞后大 → 看到车身在画面里斜转。根因是 UE 旋转 lag 非临界阻尼 + 死贴车 transform yaw。
2. **倒车车开出画**(§6)：倒车 = 车朝身后相机怼过去（target moving toward camera）+ 短臂 → 出画。

### 处方（照书）
- 朝向：改成**朝车的前进/速度方向做临界阻尼重定向**（车头转向感、不甩尾），而非 UE 那种松 lag。
- 倒车：**倒车时把臂拉长/抬高**（或按速度方向重定位），保证车在画面内 + 位置用临界阻尼防过冲。
- 预测：look-at 沿运动方向**提前一点**，让玩家看到前方（加阻尼防急转猛动）。

### 调参速查
| 现象 | 调什么 | 方向 |
|---|---|---|
| 转向晕/镜头甩太快 | 旋转跟随速度 | 调小（更柔） |
| 甩尾感（车身斜转） | 旋转跟随速度 | 调大（更跟车头） |
| 倒车丢车 | 倒车臂长缩放 / 抬高 | 调大 |
| 镜头像焊在车上(僵) | 弹簧/阻尼 | 调软一点 |
| 镜头远/近 | TargetArmLength | 增/减 |
| 看不到前方 | look-ahead 预测距离 | 调大 |

> 核心一句话：**相机别死贴车的 transform——贴"它要去哪(速度方向)"，用临界阻尼平滑地追，并永远把车留在画面里。**
